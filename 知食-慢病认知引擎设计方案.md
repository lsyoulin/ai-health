# 知食 · 慢病认知引擎设计方案

> 让每一次饮食分析，都成为一次自然的学习

---

## 一、设计哲学：不做百科，做场景化认知

### 1.1 核心问题

慢病患者存在巨大的认知鸿沟：
- 67% 的糖尿病患者不了解 HbA1c 控制目标
- 80% 的高血压患者不知道"限盐 5g/日"具体多少
- 大多数患者不了解并发症预警信号
- 低血糖急救知识普及率 <30%

**但传统健康 App 的疾病百科没人看**——静态知识库与用户当前场景无关，跳出率极高。

### 1.2 设计原则

**反百科**：不做静态知识库，所有疾病知识都从"当前这顿饭"自然延伸

**场景触发**：用户在分析食物时，AI 在恰当 context 主动带出相关知识

**因果链教育**：不只告诉用户"血糖会升到 11.2"，更告诉用户"长期餐后血糖 >10 会增加视网膜病变风险 35%"——让用户理解"为什么"而不只是"是什么"

**每日轻量**：每天一个 30 秒可读完的知识卡片，基于用户 Persona 个性化推送

**急救场景嵌入**：低血糖/高血糖危象识别自然嵌入饮食分析流程

### 1.3 一句话定位

> **每次吃饭，都是一次理解自己身体的机会。**

## 二、五大融合机制

### 2.1 机制一：饮食-疾病因果链（核心）

**触发**：每次食物分析时
**形式**：在预测结果下方自然延伸"为什么重要"

**示例**：

```
分析：红烧肉 + 米饭
预测：餐后血糖 11.2 mmol/L（超标）

━━━ 为什么这对您重要 ━━━

长期餐后血糖 >10 mmol/L 会加速血管损伤
├─ 视网膜病变风险 ↑35%
├─ 糖尿病肾病风险 ↑28%
└─ 心血管事件风险 ↑19%

→ 调整方案：减 1/3 米饭 + 先吃青菜
→ 预测血糖：7.8 mmol/L（安全）
→ 视网膜病变风险年累积 ↓8%
```

**设计要点**：
- 因果链数据基于权威医学研究（ADA、中华医学会糖尿病分会等）
- 数字化呈现风险变化，让抽象疾病变具体
- 优化方案不只是降血糖，更是降"风险"
- 让用户每次吃饭都看见自己在守护身体

### 2.2 机制二：食物即课程

**触发**：每次分析新食物时
**形式**：附带"今天学到一课"模块

**示例**：

```
分析：咸鸭蛋（高血压患者）

━━━ 今天学到 ━━━

一个咸鸭蛋含盐 ≈ 4g
您今日已摄入钠 2.3g
WHO 建议：每日钠 <2g（≈5g 盐）

为什么高血压要限钠？
钠会让身体保留水分 → 血容量增加 → 血压升高
长期高钠饮食会降低降压药效果

→ 今日剩余钠额度：已超标
→ AI 建议：今日其他餐次避免高盐食物
```

**设计要点**：
- 每个食物附带一个相关知识卡片
- 从"这个食物有什么营养"延伸到"这个食物对您疾病意味着什么"
- 知识卡片可累积，形成用户的"个人饮食知识图谱"

### 2.3 机制三：并发症预警雷达

**触发**：基于用户长期饮食数据分析
**形式**：周报/月报中的风险趋势可视化

**示例**：

```
本周风险雷达：

  视网膜病变 ●●●○○ 中风险
  糖尿病肾病 ●●○○○ 低风险
  心血管事件 ●●●●○ 高风险
  神经病变   ●●○○○ 低风险

主要影响因素：
本周 5 次餐后血糖 >10，3 次钠摄入超标

→ 长期趋势如何？
→ 您本月平均餐后血糖 9.8（vs 上月 10.5，↓7%）
→ 心血管事件风险月度下降 12%

→ 推荐学习：
《糖尿病视网膜病变的早期信号》3 分钟阅读
```

**设计要点**：
- 不是静态风险，而是动态趋势——让用户看见改善
- 与饮食数据直接关联，"吃什么"影响"什么风险"
- 风险卡片附带学习入口，自然引导深入了解

### 2.4 机制四：低血糖/高血糖急救知识嵌入

**触发**：检测到用药+饮食冲突，或用户主动查询
**形式**：红色警示卡 + 急救流程

**场景 A：低血糖预警**

```
分析：用户记录了胰岛素注射，但本次餐食碳水仅 15g

⚠️ 低血糖风险预警

您本次餐食碳水过低，注射胰岛素后可能出现低血糖
低血糖症状：心慌、出汗、手抖、饥饿感、严重可昏迷

━━━ 15 法则急救 ━━━

1. 立即进食 15g 快速糖
   - 3 块水果糖
   - 半杯橙汁
   - 1 汤匙蜂蜜

2. 等待 15 分钟复测血糖
3. 若仍 <3.9 mmol/L，重复上述步骤
4. 缓解后加餐一份复合碳水稳住血糖

→ 建议：本次餐食增加 30g 碳水（如半片面包）
→ 预测血糖：6.5 mmol/L（安全）
```

**场景 B：高血糖危象识别**

```
分析：用户记录血糖 18 mmol/L

⚠️ 高血糖危象预警

血糖 >16.7 mmol/L 有酮症酸中毒风险
预警症状：口干、多尿、乏力、恶心、呼吸深快（烂苹果味）

━━━ 紧急处理 ━━━

1. 立即测酮体（如有试纸）
2. 大量饮水稀释血糖
3. 按医嘱追加胰岛素
4. 若出现呕吐/腹痛/意识模糊 → 立即就医

→ 不是饮食能解决的问题，需要医疗介入
```

**设计要点**：
- 急救知识在"最需要的时候"出现，而不是百科里被动查找
- 15g 糖原则、酮症酸中毒识别——这些是真能救命的知识
- 同时给出饮食调整建议（治标）+ 医疗建议（治本）

### 2.5 机制五：每日一课（轻量推送）

**触发**：每日首次打开 App
**形式**：3 秒可读完的卡片，可滑过

**示例**：

```
今天一课 · 糖尿病篇

"先菜后饭"为什么有效？
先吃蔬菜 → 胃里有纤维铺垫 →
减缓糖分吸收速度 →
餐后血糖峰值降低 35%

→ 您今天的午餐：先青菜 → 后米饭
→ 预测血糖 7.8（vs 直接吃米饭 10.2）

[ 我知道了 ]
```

**设计要点**：
- 30 秒内可读完
- 基于用户 Persona 个性化（糖尿病/高血压不同内容）
- 与用户实际行为关联（如今天的午餐就是"先菜后饭"）
- 不强迫阅读，自然下滑即关闭

## 三、产品交互整合

### 3.1 在现有页面的嵌入位置

| 现有页面 | 嵌入内容 | 嵌入位置 |
|---------|---------|---------|
| 首页 | 每日一课卡片 | 预测环下方 |
| 分析页 | 因果链教育 + 食物即课程 | 预测结果下方 |
| Persona 页 | 同一菜品对不同并发症影响对比 | Persona 切换器下方 |
| 推演页 | "调整后风险降低 X%"动态展示 | 优化结果对比区 |
| 父母页 | 为父母学到的一课 + 并发症预警 | 父母分析结果下方 |
| 教练页 | 主动查询疾病知识入口 | 对话教练新增"知识"模式 |
| 趋势页 | 并发症预警雷达 | 月度趋势报告 |

### 3.2 新增页面：慢病认知中心（可选）

`/learn` — 系统化学习路径

**不作为核心页面**，而是作为已有页面的延伸：
- 用户在分析页学到某个知识 → 可点击"深入了解"跳转到认知中心
- 认知中心按疾病类型 + 主题分类组织
- 包含：
  - 疾病基础（糖尿病/高血压是什么）
  - 饮食管理（为什么这么吃）
  - 并发症预防（长期风险与预警信号）
  - 急救知识（低血糖/高血糖危象处理）
  - 用药与饮食（药物与食物相互作用）

### 3.3 数据结构

```typescript
// 每个食物附带知识卡片
interface FoodKnowledgeCard {
  foodId: string
  personaType: PersonaType
  title: string           // "为什么高血压要限钠"
  content: string         // 30 秒可读完
  relatedRisk: string     // 关联风险"心血管事件"
  source: string          // 来源 "WHO 2023 钠摄入指南"
}

// 因果链数据
interface CausalChain {
  metric: 'glucose' | 'bloodPressure' | 'sodium' | 'fat'
  threshold: number      // 阈值
  riskType: string       // "视网膜病变"
  riskIncrease: number   // "+35%"
  source: string         // 研究来源
}

// 并发症风险雷达
interface RiskRadar {
  personaType: PersonaType
  risks: {
    name: string         // "视网膜病变"
    level: 1-5          // 风险等级
    trend: 'up' | 'down' | 'stable'
    mainFactors: string[] // 主要影响因素
  }[]
}

// 急救预案
interface EmergencyProtocol {
  trigger: 'hypoglycemia' | 'hyperglycemia' | 'hypertensive_crisis'
  symptoms: string[]
  steps: string[]
  dietaryAdvice: string
  medicalAdvice: string
}
```

### 4.2 内容生产原则

1. **权威来源**：ADA、AHA、中华医学会糖尿病/高血压分会、WHO 指南
2. **每条卡片 <100 字**：30 秒可读完
3. **数字化呈现**：用具体数字而非抽象描述
4. **与饮食关联**：每条知识都可关联到具体食物或饮食行为
5. **个性化推送**：基于用户 Persona + 近期饮食数据推送

## 六、MVP 落地方案

### 6.1 优先级

| 优先级 | 功能 | 工作量 | 价值 |
|--------|------|--------|------|
| P0 | 因果链教育（机制一） | 中 | 核心差异化 |
| P0 | 食物即课程（机制二） | 中 | 高频触达 |
| P1 | 每日一课（机制五） | 低 | 用户留存 |
| P1 | 急救知识嵌入（机制四） | 中 | 救命价值 |
| P2 | 并发症预警雷达（机制三） | 高 | 长期价值 |
| P2 | 认知中心页面（/learn） | 中 | 系统化学习 |

### 6.2 MVP 范围（参赛 Demo 版本）

聚焦机制一、二、五——这 3 个机制最容易在 Demo 中展示且与"食"结合最自然。

**MVP 实现**：

1. **扩展数据模型**：
   - 在 `foods.ts` 中为每个食物增加 `knowledgeCard` 字段
   - 在 `healthEngine.ts` 中增加 `causalChain` 因果链计算

2. **扩展预测结果**：
   ```typescript
   interface AnalysisResult {
     // 现有字段...
     predictedGlucose: number
     riskLevel: 'safe' | 'warning' | 'danger'

     // 新增字段
     causalChain?: CausalChain[]      // 因果链
     knowledgeCard?: KnowledgeCard    // 食物即课程
     optimizedPlan?: OptimizationPlan // 优化方案
   }
   ```

3. **新增组件**：
   - `<CausalChainCard />` 因果链展示卡
   - `<KnowledgeCard />` 食物知识卡
   - `<DailyLesson />` 每日一课卡

4. **页面嵌入**：
   - 首页：每日一课（预测环下方）
   - 分析页：因果链 + 食物知识卡（预测结果下方）
   - Persona 页：同一菜品对不同并发症影响对比

## 七、评委冲击力

### 7.1 三个维度的差异化

| 维度 | 现有产品 | 知食 |
|------|---------|------|
| 饮食干预 | 记录你吃了什么 | 预测+优化你怎么吃 |
| 疾病教育 | 静态百科没人看 | 场景化嵌入，每次吃饭自然学习 |
| 风险管理 | 不存在 | 长期饮食数据 → 并发症风险雷达 |

### 7.2 评委记忆点

1. **"每次吃饭都是一次学习"** —— 教育不突兀，自然嵌入
2. **"调整饮食 = 降低视网膜病变风险 8%"** —— 把抽象风险变具体
3. **"低血糖 15 法则"** —— 救命知识在最需要时出现

### 7.3 AI 不可替代性再升级

- 现有产品：查表式营养数据
- 知食：基于 context 主动推理该展示什么知识
- 只有 AI 能判断"这个用户此刻最需要知道什么"
