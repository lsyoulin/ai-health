## 四、创意方向定义

### 创意名称
**知食**

### 品牌 Slogan
`知食 · 「吃」掉慢病，让每一口都被懂`

### 副标题
拍一张饭菜，AI告诉你这顿饭对你的身体意味着什么

### 品牌命名理由（评委角度）
- **文化锚点**："知食分子"是中国人国民梗，一秒记住，自带传播基因
- **双关张力**：知=AI智能，食=场景，"知食"=懂食物的健康后果
- **温度感**：从"知"到"懂" — AI不只是识别，而是懂你的身体
- **品牌延展**：知食AI(产品) / 知食分子(用户身份IP) / 知食研究所(内容专栏)
- **差异化**：避开"健康""AI""饮食"等同质化词汇堆砌
- **品类识别精准**：避免被误判为美食推荐类（"好食"风险），"知"字精准传达健康AI定位

### 一句话定位
拍一张饭菜照片，AI告诉你"这顿饭对你的血糖/血压意味着什么"

### 核心理念
**"别人帮你记录饮食，我们帮你预测饮食的健康后果"**

### 解决的问题
- 中国1.4亿糖尿病患者、3.3亿高血压患者
- 饮食是慢病管理的第一道防线，但也是最难守的
- 患者不知道"这碗牛肉面我能不能吃"、"吃了血糖会升多少"
- 营养师太贵且不可能随时在身边

### AI为什么不可替代
- 实时食物识别+营养分析：拍照即知热量、碳水、脂肪、钠含量
- **个性化健康影响预测**：同样是牛肉面，对糖尿病患者和对高血压患者，AI给出的建议完全不同
- 对话式饮食coaching："我刚才吃了碗牛肉面" → AI："根据您的糖尿病profile，预计餐后血糖会升至11.2mmol/L，建议午餐减少主食，晚餐增加蔬菜"
- 趋势分析："本周碳水摄入偏高，血糖波动较大，建议下周调整主食结构"

### 差异化定位

| 别人做的 | 我们做的 |
|---------|---------|
| 记录你吃了什么 | **预测这顿饭对你的健康意味着什么** |
| 提供热量数据 | **给出个性化健康影响评估** |
| 被动记录工具 | **主动对话式coaching** |
| 面向减肥人群 | **面向慢病患者（1.4亿+3.3亿）** |

## 六、产品方案

### 6.1 核心体验（3秒惊艳）

评委体验Demo的时间可能只有1-2分钟。必须3秒内抓住注意力：

1. **打开即看到**：一张诱人的饭菜照片 + AI分析结果
2. **1次交互即惊艳**：点击"分析我的午餐"→ 拍照/选图 → AI 3秒出结果
3. **1段对话即理解**：AI主动说"根据您的糖尿病情况，这顿饭预计让餐后血糖升至11.2mmol/L，建议..."

### 6.2 三个核心场景

**场景1：拍照分析**
- 拍一张饭菜照片 → AI识别食物种类和分量
- 分析营养成分（热量、碳水、脂肪、蛋白质、钠）
- 基于用户慢病profile预测健康影响
- 给出个性化建议（如"预计餐后血糖升至11.2mmol/L，建议减少主食摄入"）

**场景2：对话式coaching**
- 用户："我刚才吃了碗牛肉面"
- AI："根据您的糖尿病profile，这碗牛肉面约含75g碳水，预计餐后血糖会升至11.2mmol/L。建议午餐不再额外摄入主食，晚餐增加绿叶蔬菜以平衡血糖。另外，面汤含钠较高，对您的高血压也需注意。"
- 支持多轮对话，持续追问

**场景3：趋势分析**
- 本周饮食营养趋势图表
- AI洞察："本周碳水摄入偏高，血糖波动较大，建议下周调整主食结构"
- 健康评分和改进建议

### 6.3 用户慢病Profile设计

```
慢病类型：2型糖尿病 + 高血压
确诊时间：2023年
当前用药：二甲双胍 500mg bid
空腹血糖：7.2 mmol/L
餐后2小时血糖：9.8 mmol/L
血压：135/85 mmHg
身高/体重：170cm / 75kg (BMI 26.0)
饮食偏好：北方口味，偏咸
过敏/禁忌：无
```

### 6.4 Demo形态
**纯前端HTML单页应用**（符合初赛要求，不需要后端服务器）
- 体验地址或HTML文件上传

### 6.5 核心亮点设计（评委惊艳点）

> 站在评委1-2分钟体验时间窗口内，亮点必须满足：3秒惊艳 + 没有AI做不了 + 情感冲击。

#### 亮点 1（必做 · 3秒惊艳）：餐后血糖预测环

**交互流程**：拍照后3秒内，屏幕中央炸出一个动态可视化。

**视觉构成**：
- 一个圆环显示 **"预测餐后2小时血糖值：11.2 mmol/L"**
- 红黄绿三色信号灯（红色=超标 / 黄色=临界 / 绿色=安全）
- 旁边并列显示 **两个对比曲线**："不调整 vs 建议调整后"
- 一句话总结："这碗牛肉面，预计让您的血糖飙升3.4 mmol/L"

**为什么不可替代**：这是市面上没人做的"预测"瞬间。其他App只告诉你"75g碳水"，我们告诉你"11.2 mmol/L + 怎么救"。

---

#### 亮点 2（必做 · AI不可替代证明）：慢病 Persona 切换器

**交互流程**：评委可一键切换慢病身份，**同一道菜看3种AI建议**。

**三种 Persona 场景**：
- 切换"2型糖尿病" → AI关注血糖曲线与碳水负担
- 切换"高血压" → AI关注钠含量与血压波动
- 切换"糖尿病+高血压" → AI给出复合建议（同时考虑血糖+钠）

**为什么不可替代**：传统营养App给所有人的建议是一样的。**只有AI能基于个性化Profile动态推理**——这是评委一眼就能感知的"AI不可替代性"。

---

#### 亮点 3（杀手锏 · 医疗级AI感）：饮食决策推演

**交互流程**：评委可拖动滑块做"如果...则..."推演。

**推演示例**：
- "如果减少1/3米饭 → 餐后血糖 8.5 mmol/L（↓2.7）"
- "如果餐后散步20分钟 → 血糖峰值降低1.2 mmol/L"
- "如果换成糙米 → 血糖反应降低35%"
- "如果搭配1份西兰花 → 血糖曲线更平缓"

**为什么不可替代**：这是医疗级AI推理能力——把营养学知识+慢病病理学+用户Profile实时联动的复合计算，传统App根本做不到。

---

#### 亮点 4（情感杀手锏）：为父母分析模式

**交互流程**：评委大概率不是慢病患者，但家里有慢病父母。Demo首屏提供"为父母分析"入口。

**交互路径**：
- 输入父母的慢病情况（如：父亲，62岁，2型糖尿病+高血压）
- 拍一张父母常吃的菜（或选择预置场景）
- AI输出："对您父亲(2型糖尿病)而言，这顿饭..."

**为什么不可替代**：瞬间引发情感共鸣 — 评委会想"我能不能给我爸用"。这是评委从"评估Demo"切换到"我想用"的关键转换点。

---

#### 亮点优先级矩阵

| 亮点 | 必做度 | 价值 | 开发成本 | 备注 |
|------|--------|------|---------|------|
| 餐后血糖预测环 | ★★★★★ | 3秒惊艳 | 中 | Demo 第一印象决定生死 |
| 慢病Persona切换 | ★★★★★ | AI不可替代证明 | 中 | 评委一眼可见的"AI能力" |
| 饮食决策推演 | ★★★★ | 医疗级AI感 | 高 | 提升专业感，差异化加分 |
| 为父母分析模式 | ★★★★★ | 情感共鸣转换点 | 低 | 评委家庭代入，传播加速 |

**Demo核心三件套**：预测环 + Persona切换 + 为父母分析 — 这三个就足以打动评委。

## 七、技术实现路径

### 7.1 技术架构

| 功能模块 | 实现方式 | 可行性 |
|---------|---------|--------|
| 食物拍照识别 | 调用AI视觉API（百度AI/TRAE内置） | 高 |

| 趋势分析 | 前端图表 + 模拟数据 | 高 |

### 7.2 关键技术决策点

1. **食物识别**：是用真实AI视觉API，还是用预置图片+模拟识别结果？
   - 建议：Demo阶段用预置图片+模拟识别，保证体验流畅
   
2. **LLM对话**：用哪个LLM API？
   - TRAE内置AI能力 / MiniMax / OpenAI
   - 需要注意不要暴露供应商名称

3. **数据存储**：Demo阶段用localStorage即可，不需要后端数据库

### 变更 011 — W5-6 饮食组合优化器后端算法实现
- **变更时间**：2026-07-15
- **变更类型**：核心算法实现
- **决策背景**：W5-6 里程碑要求实现饮食组合优化器（贪心 + 规则约束），这是知食的核心差异化亮点，从"预测"到"优化"
- **算法设计**：
  1. **预测阶段**：基于食物组合（多食物 + 分量）+ Persona 计算餐后血糖/血压影响
     - 血糖增量 = (碳水g × GI/100 × 敏感度) / 10
     - 血压增量 = (钠mg / 1000 × 敏感度) × 2.2
     - 加权 GI 计算（按碳水含量加权平均）
     - 血糖负荷 GL = GI × 碳水g / 100
  2. **优化阶段**：贪心 + 6 条医学规则约束
     - 规则1：碳水超标 → 减少主食分量 1/3
     - 规则2：GI 过高 → 替换为低 GI 主食（降 35%）
     - 规则3：钠超标 → 减少调味品/不喝汤
     - 规则4：蔬菜不足 → 增加蔬菜 150g
     - 规则5：热量超标 → 餐后运动 20 分钟
     - 规则6：进餐顺序 → 先菜后肉再主食（降 25%）
  3. **Persona 敏感度系数**：
     - 糖尿病：血糖敏感度 1.4，钠敏感度 0.8
     - 高血压：血糖敏感度 0.8，钠敏感度 1.5
     - 糖尿病+高血压：血糖敏感度 1.5，钠敏感度 1.4
     - 健康人：血糖敏感度 0.6，钠敏感度 0.6
  4. **综合评分**：血糖(50%) + 血压(30%) + 均衡度(20%)
- **API 设计**：
  - `POST /api/optimize` — 分析并优化一餐（输入：personaId + items[{foodId, amountG}]，输出：预测 + 优化建议 + AI建议）
  - `POST /api/optimize/simulate` — 决策推演（5 个可调变量：减少碳水/增加蔬菜/运动/替换主食/调整顺序）
- **新增文件**：
  - `server/src/services/optimizer.ts` — 组合优化器核心算法（370+ 行）
  - `server/src/routes/optimize.ts` — 优化器 API 路由
- **修改文件**：
  - `server/src/index.ts` — 注册 optimize 路由
- **验证结果**：
  - TypeScript 编译通过
  - API 测试：白米饭 150g + 红烧肉 100g + 菠菜 100g（糖尿病 Persona）
    - 预测血糖 12.7 mmol/L（danger），GI 78，碳水 51.8g
    - 优化后血糖 8.2 mmol/L（warning，降 4.5）
    - 3 条优化建议：替换低GI主食/增加蔬菜/调整进餐顺序
  - 决策推演测试：5 个变量全开
    - 推演后血糖 7.4 mmol/L（从 12.7 降 5.3，safe）
    - 碳水 37.8g，GI 40，运动降糖 1.2 mmol/L
